Common Design

The Common Design project aims to provide a visual framework that will help:

  • Navigate seamlessly accross OCHA services and platforms
  • Improve users' understanding of all digital services provided by OCHA and their relations to each other
  • Improve user experience, efficiency and satisfaction
  • Create a systematic structure that allows future additions with minimal design support

Implementation notes

Browser support:

Full support: Chrome (Windows, Mac, iOS, Android), Firefox (Windows, Mac, iOS, Android), Safari (Mac, iOS), IE11, Opera (Windows, Mac), Samsung Internet v6+ (Android)

Functional support: IE10, IE9, IE8, Opera Mini (Android), Android Browser, Samsung Internet v5.

The following Modernizr detects are required for full browser support: mediaqueries, svg, cssgrid & cssgridlegacy.

Autoprefixer is also required for full browser support.

Dropdowns

The demo uses Bootstrap dropdown, if your theme already has dropdown functionality adapt the header to use that instead.

Icons

An icon font generated with Icomoon is used in the demo. These icons can be included with your theme's existing icon system instead of being included seperately. The icons are in common-design/icons/.

Namespacing

All the classes are prefixed 'cd-' to prevent clashes with existing theme styles.

Sass & CSS

Sass is used for the Common Design, the Sass files from /common-design/sass/ can be added directly to your theme. See /common-design/css/styles.scss for the order the files should be included in and notes in the individual files on their usage.

If you are not using Sass, the generated CSS can be downloaded from /common-design/css/styles.css
